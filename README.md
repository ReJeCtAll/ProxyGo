# ProxyGo - 专业级Chrome代理管理工具

> 🚀 **下一代智能代理管理扩展** - 不仅仅是代理切换，更是完整的网络管理解决方案

[![Chrome Web Store](https://img.shields.io/badge/Chrome-Web%20Store-green)](https://chrome.google.com/webstore)
[![Manifest V3](https://img.shields.io/badge/Manifest-V3-blue)](https://developer.chrome.com/docs/extensions/mv3/)
[![License](https://img.shields.io/badge/License-MIT-yellow)](LICENSE)

ProxyGo 是一款功能强大的Chrome浏览器扩展，专为需要精细化代理管理的用户设计。它不仅提供基础的代理切换功能，更集成了网络分析、规则管理、诊断工具等专业特性，让代理使用变得智能化和可视化。

## ✨ 核心特性

### 🎯 智能代理规则引擎
- **多模式支持**：白名单模式（匹配直连）和黑名单模式（匹配代理）
- **灵活规则语法**：支持精确匹配、通配符、子域名、IP地址段
- **多规则源**：手动编辑、远程URL、文件导入三种规则来源
- **实时生效**：规则修改立即应用，无需重启浏览器

### 📊 实时网络透视板
- **访问统计**：TOP10域名访问排行，实时请求计数
- **代理状态可视化**：清晰展示哪些请求走了代理，哪些直连
- **历史记录**：最近50条请求的详细记录和时间线
- **性能监控**：域名响应时间和连接状态跟踪

### 🔧 专业诊断工具套件
- **网络连接测试**：支持代理/直连模式的URL可达性检测
- **规则冲突检测**：自动识别和修复规则间的冲突
- **规则匹配测试**：验证域名是否符合预期规则
- **连接性能分析**：响应时间、状态码、重定向跟踪

### ⚙️ 高级规则管理
- **批量操作**：导入/导出规则文件，支持常见格式
- **远程规则同步**：从URL定时更新规则列表
- **规则预览**：应用前预览和验证规则效果
- **版本控制**：规则变更历史和回滚功能

### 🎨 现代化用户界面
- **直观操作面板**：一键切换代理模式和快速访问设置
- **多标签设置中心**：代理配置、规则管理、诊断工具分类管理
- **响应式设计**：适配不同屏幕尺寸，提供最佳用户体验
- **实时状态反馈**：图标和通知即时反映代理使用状态

## 🚀 快速开始

### 安装方式

#### 方式一：Chrome Web Store（推荐）
1. 访问 [Chrome Web Store 中的 ProxyGo](https://chrome.google.com/webstore)
2. 点击"添加至Chrome"按钮
3. 确认权限后完成安装

#### 方式二：开发者模式加载
1. 下载项目源码：点击右上角 `Code` → `Download ZIP`
2. 解压到本地目录
3. 打开Chrome浏览器，进入 `chrome://extensions/`
4. 开启右上角的"开发者模式"
5. 点击"加载已解压的扩展程序"，选择解压后的文件夹
6. 安装完成！

### 基础配置

1. **配置代理服务器**
   - 点击扩展图标打开弹出菜单
   - 点击"详细设置"进入设置中心
   - 在"代理设置"选项卡中输入代理服务器地址（如：`http://127.0.0.1:7890`）

2. **选择工作模式**
   - **白名单模式**：匹配的域名走直连，其他走代理
   - **黑名单模式**：匹配的域名走代理，其他直连

3. **添加规则**
   - 在对应的规则管理页面添加域名规则
   - 支持手动输入或从URL导入规则列表

## 📖 功能详解

### 代理设置模块

**支持的代理类型：**
- HTTP代理：`http://proxy.example.com:8080`
- HTTPS代理：`https://proxy.example.com:8080`
- 支持用户名密码认证：`http://user:pass@proxy.example.com:8080`

**代理验证：**
- 自动检测代理服务器可用性
- 实时显示连接状态
- 支持代理响应时间测试

### 规则管理系统

#### 规则语法说明

| 规则格式 | 示例 | 匹配说明 |
|---------|------|----------|
| 精确匹配 | `example.com` | 仅匹配 example.com |
| 子域名匹配 | `.example.com` | 匹配 example.com 及所有子域名 |
| 通配符匹配 | `*.example.com` | 匹配子域名但不包括主域名 |
| IP地址 | `192.168.1.1` | 精确匹配IP地址 |
| IP段匹配 | `192.168.1.0/24` | 匹配整个网段 |

#### 规则来源管理

**手动规则：**
- 在文本框中直接编辑规则
- 支持多行批量输入
- 自动过滤注释和空行

**远程规则：**
- 从HTTP/HTTPS URL加载规则文件
- 支持定时自动更新
- 预览功能确保规则正确性

**导入规则：**
- 支持文本文件导入
- 兼容主流代理工具的规则格式
- 导入前提供规则预览和冲突检测

### 网络透视板

**实时监控面板：**
- TOP10访问域名排行榜，显示访问次数和最后访问时间
- 最近50条网络请求的详细记录
- 代理使用状态的可视化展示

**数据统计功能：**
- 域名访问频次统计
- 代理/直连流量比例分析
- 请求时间线和性能分析

**调试信息：**
- 扩展运行状态监控
- 数据收集过程的详细日志
- 存储空间使用情况统计

### 诊断工具套件

**网络连接测试：**
- 任意URL的可达性检测
- 支持指定是否使用代理进行测试
- 显示响应时间、状态码、重定向信息
- 可选择显示响应头和详细时序信息

**规则冲突检测：**
- 自动扫描规则中的重复和冲突项
- 提供一键修复建议
- 展示冲突详情和影响范围

**规则匹配验证：**
- 输入域名测试是否匹配当前规则
- 显示匹配的具体规则和匹配类型
- 支持批量域名测试

### 最佳实践

1. **规则优化建议**
   - 优先使用精确匹配，避免过度宽泛的通配符
   - 定期清理不再需要的规则
   - 使用规则冲突检测保持规则集的健康

2. **性能调优**
   - 控制规则数量在合理范围内（建议<1000条）
   - 将高频访问的域名规则放在前面
   - 定期查看网络透视板，优化规则配置

3. **调试技巧**
   - 使用网络透视板监控实际代理使用情况
   - 利用诊断工具验证代理配置的正确性
   - 开启调试模式查看详细的匹配日志

## 🔧 故障排除

### 常见问题

**Q: 代理设置后没有生效？**
A: 请检查代理服务器地址格式是否正确，确保代理服务运行正常，并验证规则配置。

**Q: 某些网站无法访问？**
A: 使用诊断工具测试连接，检查是否为规则匹配问题，必要时调整规则或临时禁用代理。

**Q: 扩展图标显示异常？**
A: 重新加载扩展或重启浏览器，检查是否有浏览器权限限制。

**Q: 规则导入失败？**
A: 确认文件格式正确，检查规则语法，使用预览功能验证规则内容。

### 性能问题

**如果感觉扩展运行缓慢：**
- 减少规则数量，移除不必要的规则
- 清理网络统计数据，释放存储空间
- 检查是否有规则冲突影响匹配效率

### 兼容性说明

- **Chrome版本要求**：88+ （支持Manifest V3）
- **Edge浏览器**：支持基于Chromium的新版Edge
- **其他Chromium浏览器**：大部分功能兼容

## 🏗️ 技术架构

### 核心组件

- **Service Worker** (`background.js`)：后台处理代理逻辑和规则匹配

- **设置中心** (`settings.js/.html`)：完整的配置管理界面
- **网络透视板** (`network-dashboard.js/.html`)：实时监控和数据可视化
- **诊断工具** (`testing-tools.js/.html`)：网络测试和规则验证

### 技术特性

- **Manifest V3**：使用最新的扩展标准，确保长期兼容性
- **PAC脚本**：高效的代理自动配置，减少性能影响
- **响应式设计**：适配不同设备和屏幕尺寸
